public - корень сайта, здесь лежат все файлы, которые должны быть доступны из-вне
    - index.php - точка входа в приложение
    - assets
        - все статические файлы, в т.ч. css, js, img
app - это закрытая от доступа из интернета дирректория, здесь лежат все остальные скрипты, шаблоны, конфигурационные файлы
    - config - текстовые конфиги, в т.ч. база товаров, база пользователей
        - db_product.yaml - база товаров
    - controllers - пока, только классы, отвечающие за формирование разделов (на каждый раздел - отдельный класс)
        - IPageController - описание интерфейса всех контроллеров
        - CCatalogController - класс, отвечающий за формирование страницы каталога
        - CCartController - класс, отвечающий за формирования страницы с корзиной
        - CProductController - класс, отвечающий за формирование страницы конкретного товара
    - dataio - классы для обработки конфигурационных файлов
        - IConfParser, CConfParser - классы, описывающие конструкцию парсера конфигов
        - CTextConfParser, CYamlParser - классы реализующие чтение/запись данных в определенном формате
    - product - классы, отвечающие за поведение продуктов
    - router - классы для определения по строке запроса, какой раздел сайта необходимо отобразить
    - views - шаблоны (html) страниц и элементов страниц
        - product - шаблоны (html) отображения продукта на различных страницах
        - cart.php - шаблон страницы корзины
        - catalog.php - шаблон страницы каталога
	- product.php - шаблон страницы конкретного продукта
