тест настройки менеджера проектов - gulp

1)ставим node.js
2)затем через npm ставим глобально(-g) gulp. Для этого из коммандной строки вводим
	npm install -g gulp --save
( в принципе, это можно сделать после инициализации проекта)
3) далее в папке проекта инициализируем проект(npm init).Для этого в терминале(командной строке) открыть C:\www\dev>(то есть путь, где лежит папка проекта(сайта) ) и набрать npm init . Этим создается файл package.json
4) далее ставим gulp локально (из папки проекта) : npm install gulp --save
Этот ключ (--save) означает, что данные об установке запишуться в файл package.json.
После установки gulp локально создается файл package-lock.json.
5) запускаем комманду: npm install gulp-cli --save (gulp-cli это утилита, которая позволяет вам получить доступ к gulp из вашей оболочки.)
6) создаем файлик gulpfile.js  - в нем будем писать "правила"(таски) для gulp.
7) Далее устанавливаем разные плагины для gulp и записываем в gulpfile.js таски.

!!! Главное помнить из какой папки у нас запущена командная строка где мы запускаем таски для gulp
так как пути, прописанные в функциях могут быть другие

Примеры установки плагинов
а) библиотека для удаления файлов и папок
npm install del --save 

б) для конкатенации файлов
npm install gulp-concat --save
